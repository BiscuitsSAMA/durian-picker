# 🍈 挑个榴莲吧

## 两种模式

收到用户消息时，根据场景自动切换模式：

| 模式 | 触发场景 | 核心流程 |
|------|---------|---------|
| 🏪 **批量挑** | 用户拍一堆榴莲、说"帮我挑一个"、在一堆里选 | 识别→编号→评分→排序→推荐最佳 |
| 🔍 **单果鉴定** | 用户拍一个榴莲、说"看看这个" | 问三问→看图→判断干包/湿包 |

---

## 🏪 模式一：批量挑

用户拍一张包含**多个榴莲**的照片（比如超市货架、水果摊），从中挑出最好的。

### 工作流

```
用户发群拍照片
   ↓ Agent 看图识别
用多模态能力分割/定位图中每个榴莲 → 分配编号 A/B/C/D...
   ↓ 逐个评分
对每个榴莲独立评估：果型离心率、刺密度、颜色成熟度
   ↓ 综合排序
按总分从高到低排序
   ↓ 推荐
输出最佳选择 + 各榴莲评分明细
```

### 识别与编号
- 用多模态能力在一张图中找出所有榴莲
- 按从左到右/从上到下顺序分配编号（A、B、C、D...）
- 参考 [liujiaqi222/durian-helper-mini-program](https://github.com/liujiaqi222/durian-helper-mini-program) 的 YOLO 检测+编号思路

### 逐果评分

对每个编号的榴莲，分别评估：

| 维度 | 权重 | 评分依据 |
|------|------|---------|
| 果型离心率 | 30% | 越圆越好，<1.3偏圆，>1.5偏长 |
| 刺密度 | 25% | 越密越好，≤5mm密集，>10mm稀疏 |
| 颜色成熟度 | 25% | 该品种的典型成熟色 |
| 鼓包饱满度 | 20% | 外壳凸起越多→肉越多 |

### 输出格式

```
🏪 批量扫描结果（共 N 个）

🥇 最佳推荐：榴莲 C（总分 85/100）
   果型偏圆 ✅ | 刺密集 ✅ | 颜色金黄 ✅ | 鼓包饱满 ✅
   干包倾向，可入手

📊 排名明细：
   A: 72/100 — 果型偏长，刺中等
   B: 68/100 — 颜色偏青，未熟
   C: 85/100 — ⭐ 最佳
   D: 55/100 — 果型不规则，刺稀疏
```

## 🔍 模式二：单果鉴定

用户拍一个已挑好的榴莲，判断品质。

### 工作流

```
用户发单果照片
   ↓ 问三问
品种？摇起来有声音吗？捏刺能捏动吗？
   ↓ Agent 看图
果型离心率 | 刺密度+奶油尖 | 底部 | 颜色
   ↓ 加权判断
结论：干包/湿包/生包 + 熟度 + 是否达预期
```

### 判断权重

| 维度 | 权重 | 干包✅ | 湿包🟡 | 生包❌ |
|------|------|--------|--------|--------|
| 果型 | 30% | 偏圆、底平、离心率<1.3 | 圆/长均可 | 不规则、尖底、离心率>1.5 |
| 刺 | 25% | 尖硬密集(≤5mm)、奶油尖 | 钝圆有弹性 | 紧实捏不动、细长(>10mm) |
| 声音 | 25% | 撞击感/沙沙声 | 闷声/无声 | 完全无声 |
| 捏刺 | 20% | 微有弹性 | 偏软易捏 | 硬如石头 |

### 综合评分
```
总分 = 果型×30% + 刺×25% + 声音×25% + 捏刺×20%
```

### 结论范围
- ✅ **干包** — 干爽甜糯，达到预期
- 🟡 **湿包** — 软糯水润，看个人喜好
- 🔴 **生包/死包** — 不熟，未达预期
- 🟢 **熟了正好** — 今天可吃
- 🟠 **再放1-2天** — 有点生
- ⚫ **过熟变质** — 酒精味/水声

## 品种速查

| 品种 | 特征 |
|------|------|
| 金枕头🥇 | 金黄、会裂口、甜糯、新手友好 |
| 猫山王👑 | 黄绿、不裂需刀开、苦甜浓郁 |
| 黑刺🖤 | 偏绿不裂、奶油质地、顶级 |
| 干尧🫒 | 青绿、微裂、细腻高甜 |
| 托曼尼🟢 | 偏绿不裂、似猫山王籽大 |
